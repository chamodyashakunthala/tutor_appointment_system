 Tutor Appointment System - Requirements

 1. Project Overview
The Tutor Appointment System allows students to schedule tutoring sessions with tutors based on subject availability and preferred time slots. The system manages users (students and tutors), session scheduling, and appointment history.

 2. Actors
- Student: Searches for tutors, requests appointments.
- Tutor: Specifies available subjects and time slots, accepts or rejects appointments.
- Admin: Manages users and monitors the system.

 3. Functional Requirements
1. Students must be able to register and log in to the system.
2. Students should be able to search tutors by subject.
3. Students must be able to request an appointment.
4. Tutors should be able to accept or reject appointments.
5. Tutors must be able to set available time slots.
6. Admin should be able to view and manage users.
7. The system must send appointment confirmation notifications.

 4. Non-Functional Requirements
- Performance: The system should respond to user actions within 2 seconds.
- Security: Passwords must be stored securely using encryption.
- Usability: The user interface should be simple and easy to navigate.
- Scalability: The system should handle increasing users and appointments in the future.

 5. Constraints
- Users must have an internet connection.
- Appointment availability depends on tutor schedules.
